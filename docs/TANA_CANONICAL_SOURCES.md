### Important provenance rule

Across the canonical docs, the intended rule is:

- `Created by` always stores the human person node only.
- Never place the LLM/model into `Created by`.
- For daily-note node types that support it, also set `Generated by` to the LLM or provenance mode.
- Current daily-note expectation:
  - `#task`, `#discussion`, `#meeting`, `#output`: set both `Created by` and `Generated by`
  - `#reply`: set `Author` for the human person; no `Generated by` field yet
  - `#annotation-run`: set `Created by`; add `Generated by` only if the schema gains that field later
